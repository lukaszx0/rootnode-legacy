Managing FTP accounts

Account

Add

Add command takes 3 mandatory arguments: user name, password and directory path and optional privilege arguments. It is a good idea to generate password with apg command or following Perl code.

PASSWORD=$(perl -le 'print map { ("a".."z", 0..9)[rand 36] } 1..12')
satan ftp add example $PASSWORD ~/web/example.net/htdocs nodelete nomkdir

All privileges are enabled by default. You can disable privileges with following arguments:

  • nomkdir - forbid directory creation
  • nodelete - forbid file removal
  • noupload - forbid file upload
  • noread - forbid directory read
  • nossl - do not force encrypted connection and data transfer
Privilege argument in satan ftp add command is not mandatory.

List

List command takes no arguments.

satan dns list

Results will be similar to the following:

FTP accounts (1 in total)

User name   Directory                               mkdir   delete   upload   read   ssl   Created at            Updated at            
example     /home/tester/web/example.net/htdocs     yes     yes      yes      yes    yes   2012-10-07 09:51:50   2012-10-07 09:51:50

Delete

Delete command takes 1 argument: user name.

satan ftp del example

Notice FTP account files will NOT be deleted after account removal.

Client configuration

Connection port is $UID1. For $UID = 1234 it would be 12341.
